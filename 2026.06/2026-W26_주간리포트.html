A decoder-only foundation model for time-series forecasting🔬 심층

Abhimanyu Das, Weihao Kong, Rajat Sen 외 1명

▲ 34💬 1★ 24.8k
time-series-foundation-modelzero-shotautoregressive-objectiveforecasting

패치드-디코더 어텐션으로 사전학습해, 추가 학습 없이도 다양한 시계열을 예측하는 파운데이션 모델.

NLP의 LLM 발전에 영감받아, 데이터셋마다 따로 학습하지 않고도 즉시(zero-shot) 쓸 수 있는 시계열 예측 파운데이션 모델이 필요했다. 기존엔 데이터셋별 지도학습 모델이 각각 최고 성능을 냈다.

대규모 시계열 코퍼스에 patched-decoder 스타일의 어텐션 모델을 사전학습한다. 다양한 관측 길이·예측 길이·시간 granularity에 두루 동작하도록 설계해, 공개 데이터셋에서 제로샷 성능이 데이터셋별 SOTA 지도학습 모델에 근접한다.

  • 시계열 예측용 디코더 전용(decoder-only) 파운데이션 모델 제안
  • 여러 공개 데이터셋에서 제로샷 성능이 데이터셋별 SOTA 지도학습에 근접
  • 관측·예측 길이와 시간 granularity 변화에 두루 일반화
  • 코드 공개(Google, ★24.8k)

방법 상세

디코더 전용(decoder-only) 트랜스포머에 '패치드-디코더' 어텐션을 쓴다. 입력 시계열을 겹치지 않는 패치(입력 패치 32 timepoints)로 잘라 각 패치를 residual block으로 토큰화한 뒤 causal self-attention으로 처리한다. 핵심 설계는 출력 패치(128 timepoints)를 입력 패치보다 길게 둔 점 — 예: 256 구간 예측에 자기회귀 단계가 8회가 아닌 2회로 줄어 추론이 효율적이다. 모델 차원 1280, 20개 레이어·16 헤드, 약 200M 파라미터(17M·70M 어블레이션 포함).

실험 셋업

  • 제로샷 평가 3대 벤치마크군: Monash(18개 데이터셋), Darts(8개 단변량), ETT(4개, horizon 96/192)
  • 비교군: DeepAR·PatchTST·FEDformer/Autoformer/Informer·ARIMA·llmtime(GPT-3 기반)
  • 학습: 모든 패치에 대한 MSE 손실 + 가변 컨텍스트 노출용 랜덤 마스킹(매 스텝 0~p-1 만큼 앞부분 마스킹 → 1~최대 길이 모두 학습)

데이터셋

  • 사전학습 코퍼스(실데이터+합성 약 50:50) — 약 1000억 timepoints (Google Trends(~22k 쿼리, 2007~2022) · Wikipedia 페이지뷰(2012~2023) · M4(~10만 시계열) · Traffic/Electricity · 합성 300만 시계열×2048)

한계 · 비판

  • 원문에 명시적 '한계' 절은 두드러지지 않음 (저자 인정)
  • 점(point) 예측 위주 — 확률적(probabilistic) 예측은 '쉽다'고만 언급, 미검증 (추가 의문)
  • OOD 제로샷이 왜 잘 되는지에 대한 이론적 근거는 부족 (추가 의문)
  • 평가가 3개 벤치마크군에 한정, 실패 모드·취약 도메인 분석 없음 (추가 의문)

재현 가능성

코드 공개가중치 공개데이터 부분라이선스 Apache-2.0(레포 기준)

원논문(2023)엔 공개 언급이 없었으나, 이후 google-research/timesfm로 코드·가중치 공개(★24.8k)

📌 내 학습 포인트

  • 출력 패치를 입력보다 길게 둬 자기회귀 단계 수를 줄이는 효율 트릭 — 다른 생성 과제에도 응용 가능
  • 랜덤 마스킹으로 1~최대까지 모든 컨텍스트 길이를 학습 → 가변 입력 강건성 확보
  • 200M로 GPT-3(llmtime) 대비 ~1/1000 규모에서 더 나은 제로샷 — '도메인 특화 사전학습 > 범용 거대모델'의 사례

연결 연구

  • ← llmtime (GPT-3로 시계열 예측): LLM을 직접 예측에 쓴 선행 — TimesFM이 능가
  • ← PatchTST: 패치 기반 시계열 트랜스포머(지도학습 강baseline)
  • → Kronos (지난주 W25): 같은 흐름의 금융 특화 시계열 파운데이션 모델

Claude 종합 코멘트

출력>입력 패치, 랜덤 마스킹, 100B 규모 실+합성 코퍼스라는 단순하지만 효과적인 설계로 '시계열의 GPT'를 만든 기념비적 연구다. 지난주 Kronos와 함께 보면 시계열 파운데이션 모델이 도메인(범용→금융)을 넓혀가는 흐름이 뚜렷하다. 다만 점 예측 한정·이론적 근거 부족은 남는 숙제.

오늘날 에이전트 스킬은 수작업, 일회성 생성, 느슨하게 통제된 자기수정으로 만들어져 시작점 대비 일관되게 개선되지 않는다. 가중치 최적화처럼 재현 가능하고, 피드백 하에서 확실히 향상되는 '스킬 학습' 방법이 없었다.

프리즈된(고정된) 에이전트의 외부 상태로서 스킬 문서를 학습한다. 별도의 옵티마이저 모델이 점수화된 롤아웃을 보고 단일 스킬 문서에 add/delete/replace 편집을 제안하되, 보류 검증 점수를 '엄격히 향상'시킬 때만 편집을 수용한다. 텍스트 학습률 예산·거부 편집 버퍼·에폭 단위 slow/meta 업데이트로 학습을 안정화하며, 배포 시 추가 추론 호출은 0이다.

  • 체계적이고 제어 가능한 최초의 텍스트 공간(text-space) 스킬 옵티마이저 제안
  • 6개 벤치마크·7개 모델·3개 실행 환경(직접 대화·Codex·Claude Code)의 전 52개 셀에서 최고 또는 동률

  • 최적화한 스킬이 모델 규모·실행 환경·인접 수학 벤치마크로 재최적화 없이 전이됨

방법 상세

옵티마이저 모델이 점수화된 롤아웃을 입력받아 단일 스킬 문서에 경계가 있는(bounded) add/delete/replace 편집을 생성한다. 각 편집은 보류(held-out) 검증 점수를 '엄격히 향상'시킬 때만 채택되어, 확률적·휴리스틱이 아닌 결정론적 수용 기준을 갖는다. 텍스트 학습률 예산이 단계별 편집 범위를 제한하고, 거부된 편집은 버퍼에 저장해 중복 탐색을 막으며, 에폭 단위 slow/meta 업데이트로 점진적·안정적으로 수렴한다. 최적화는 오프라인에서 일어나 배포 시 추가 모델 호출이 없다.

실험 셋업

  • 벤치마크 6종, 타깃 모델 7종(GPT-5.5 포함)
  • 실행 환경 3종: 직접 대화 / Codex 에이전트 루프 / Claude Code
  • 비교군: 인간 작성, 일회성 LLM 생성, Trace2Skill, TextGrad, GEPA, EvoSkill

데이터셋

  • 6개 에이전트 벤치마크 + 인접 수학 벤치마크(전이 평가) — 52개 (모델,벤치,환경) 조합 (에이전트 태스크)
