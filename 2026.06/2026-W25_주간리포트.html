오늘날 에이전트 스킬은 수작업, 일회성 생성, 느슨하게 통제된 자기수정으로 만들어져 시작점 대비 일관되게 개선되지 않는다. 가중치 최적화처럼 재현 가능하고, 피드백 하에서 확실히 향상되는 '스킬 학습' 방법이 없었다.

방법 상세

옵티마이저 모델이 점수화된 롤아웃을 입력받아 단일 스킬 문서에 경계가 있는(bounded) add/delete/replace 편집을 생성한다. 각 편집은 보류(held-out) 검증 점수를 '엄격히 향상'시킬 때만 채택되어, 확률적·휴리스틱이 아닌 결정론적 수용 기준을 갖는다. 텍스트 학습률 예산이 단계별 편집 범위를 제한하고, 거부된 편집은 버퍼에 저장해 중복 탐색을 막으며, 에폭 단위 slow/meta 업데이트로 점진적·안정적으로 수렴한다. 최적화는 오프라인에서 일어나 배포 시 추가 모델 호출이 없다.

실험 셋업

  • 벤치마크 6종, 타깃 모델 7종(GPT-5.5 포함)
  • 실행 환경 3종: 직접 대화 / Codex 에이전트 루프 / Claude Code
  • 비교군: 인간 작성, 일회성 LLM 생성, Trace2Skill, TextGrad, GEPA, EvoSkill

데이터셋

  • 6개 에이전트 벤치마크 + 인접 수학 벤치마크(전이 평가) — 52개 (모델,벤치,환경) 조합 (에이전트 태스크)

한계 · 비판

  • 초록 수준에서는 명시적 한계 서술이 제시되지 않음 (저자 인정)
  • 옵티마이저 모델 자체의 품질·비용에 성능이 의존할 수 있음 (추가 의문)

KV 캐시를 GPU 밖으로 빼내 쿼리·엔진 간 재사용하는 오픈소스 캐시 계층(최대 15배 처리량).

KV 캐시는 보통 GPU 메모리에 저장돼 디코딩을 가속하지만, 쿼리·엔진 간 재사용을 위해 GPU 밖으로 옮길 필요가 커졌다. 사용자 KV 캐시 총량이 GPU 용량을 빠르게 초과했음에도 효율적인 오프로딩·전송 솔루션이 없었다.

vLLM·SGLang이 생성한 KV 캐시를 GPU 메모리 밖으로 추출·저장해 엔진과 쿼리 간 공유한다. 캐시 오프로딩(쿼리 간 프리픽스 재사용)과 prefill-decode 분리(엔진/GPU 간 전송)를 지원하며, 배치 데이터 이동·연산/IO 파이프라이닝, 모듈식 커넥터, 캐시 오케스트레이션 제어 API를 제공한다.

  • 최초이자 가장 효율적인 오픈소스 KV 캐시 솔루션
  • vLLM 결합 시 다회 QA·문서 분석 등에서 최대 15배 처리량 향상
  • 엔진 진화와 분리되는 모듈식 KV 캐시 커넥터 + 제어 API
  • 엔터프라이즈 실사용 인사이트(예: 컨텍스트 절단이 프리픽스 캐시 적중률을 절반으로 떨어뜨림)
