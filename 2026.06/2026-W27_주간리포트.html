Unlimited OCR Works🔬 심층

Youyang Yin, Huanhuan Liu, YY 외 14명

▲ 38💬 5★ 11.5k
end-to-end-ocrkv-cachesliding-window-attentiondocument-parsing

상수 KV 캐시(R-SWA)로 수십 페이지 문서를 한 번에 받아 적는, 인간 작업기억을 모사한 OCR 모델.

DeepSeek OCR처럼 LLM을 디코더로 쓰는 엔드투엔드 OCR은 언어의 사전분포를 활용해 성능이 좋지만, 출력이 길어질수록 KV 캐시가 누적돼 메모리가 늘고 생성이 느려진다. 긴 필사에도 효율이 떨어지지 않는 인간과 대조된다.

DeepSeek OCR을 베이스라인으로, 디코더의 모든 어텐션을 제안 기법 R-SWA(Reference Sliding Window Attention)로 교체한다. R-SWA는 어텐션 연산을 줄이면서 디코딩 내내 KV 캐시를 '상수'로 유지한다. 인코더의 높은 압축률과 결합해 32K 최대 길이에서 수십 페이지를 단일 forward로 전사한다. R-SWA는 OCR뿐 아니라 ASR·번역에도 쓰이는 범용 파싱 어텐션이다.

  • 디코딩 내내 일정한 KV 캐시를 유지하는 R-SWA 어텐션 제안
  • 32K 길이에서 수십 페이지 문서를 단일 forward로 전사
  • OCR을 넘어 ASR·번역 등에 적용 가능한 범용 파싱 어텐션
  • 코드·가중치 공개(Baidu, ★11.5k)

디코더 어텐션을 R-SWA로 교체해 디코딩 내내 KV 캐시를 상수로 유지한다.

방법 상세

핵심은 R-SWA(Reference Sliding Window Attention). 디코딩 중 KV 캐시를 두 부분으로 유지한다 — 고정 프리픽스(시각+프롬프트 토큰, 길이 L_m)와 생성 텍스트에 대한 폭 n(기본 128)의 인과적 슬라이딩 윈도우. 캐시 크기는 C = L_m + min(n, T) ≤ L_m + n 으로, 윈도우 크기까지만 커지고 이후 '상수'다(표준 MHA는 L_m + T로 선형 증가). 시각(reference) 토큰은 상태 전이 없이 디코딩 내내 전역으로 접근 가능해, 일반 슬라이딩 윈도우와 달리 시각 특징이 '흐려지지' 않는다. 베이스라인 DeepSeek OCR의 인코더(DeepEncoder: SAM-ViT + CLIP-ViT, 16× 토큰 압축, 1024² → ~256 토큰)는 동결하고 디코더 어텐션만 R-SWA로 교체, LLM 파라미터만 학습한다. 디코더 3B(MoE로 ~500M 활성).

실험 셋업

  • 주 벤치마크: OmniDocBench v1.5/v1.6
  • 비교: DeepSeek OCR(베이스라인)
  • 장문 평가: 2~40+ 페이지 문서를 단일 forward로 전사

데이터셋

  • 문서 학습 샘플 — 약 200만(단일:다중 페이지 ≈ 9:1) (문서 OCR)

한계 · 비판

  • 프리필 32K 제한 — 페이지가 매우 많으면 한계(향후 128K 학습 예정) (저자 인정)
  • 다중페이지 모드는 Base 인코더(1024²)라 해상도 손실 → 20페이지 넘으면 edit distance 상승 (저자 인정)
  • 학습 샘플 200만으로 제한적 (저자 인정)
  • ASR·번역 일반화는 주장만 있고 실험 검증 없음(future work) (추가 의문)
  • KV캐시 절감이 초장문에서 품질 저하와 트레이드오프 (추가 의문)

coarse-to-fine 2단계 전략으로 전역 레이아웃 분석과 지역 내용 인식을 분리한다. 1단계는 다운샘플 이미지에서 효율적으로 구조 요소를 파악하고, 2단계는 그 레이아웃을 길잡이 삼아 원해상도 crop에서 표적 인식해 조밀한 텍스트·수식·표의 세부를 보존한다. 전용 데이터 엔진으로 사전학습·파인튜닝 코퍼스를 대규모 생성한다.

방법 상세

1.2B VLM = 비전 인코더(675M NaViT, Qwen2-VL 초기화, 2D-RoPE로 동적 해상도) + 언어모델(0.5B Qwen2-Instruct 디코더, M-RoPE) + 패치 머저(2×2 비전 토큰 pixel-unshuffle). 2단계 파이프라인: 1단계는 입력을 1036×1036 썸네일로 리사이즈해 전역 레이아웃을 저비용으로 분석하고, 2단계는 검출된 레이아웃으로 원해상도(최대 2048²) crop을 떠 조밀한 텍스트·수식·표를 인식한다. 이 분리로 엔드투엔드 원해상도 처리의 O(N²) 토큰 복잡도를 회피한다.

Geometric Context Transformer for Streaming 3D Reconstruction

Lin-Zhuo Chen, Jian Gao, Yihang Chen 외 8명

▲ 21💬 2★ 8.2k
3d-reconstructionslamgeometric-context-transformerworld-model

SLAM 원리를 담은 어텐션으로 영상 스트림에서 실시간(약 20FPS) 3D를 복원하는 피드포워드 파운데이션 모델.

스트리밍 3D 재구성은 영상 스트림에서 카메라 포즈·점군을 복원해야 하며, 기하 정확도·시간적 일관성·계산 효율을 동시에 요구한다.
